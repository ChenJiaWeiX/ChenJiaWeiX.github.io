# 一. 对象的几种封装模式

# 原始模式

示例

var cat1 = {};
 cat1.name = '小花'
 cat1.age = '1'

var cat2 = {};
 cat2.name = '小黑'
 cat2.age = '2'

# 工厂模式

说明

  • 工厂模式，是用函数实现对象封装并实例化的方法。

示例

function cat (name,age) {
     var o = {};
     o.name = name
     o.age = age;
     o.sayName = function () {
         console.log(this.name);
     }
     return o;
 }
 var cat1 = cat('小花','1')
 var cat2 = cat('小黑','2')
# 细节
  • 工厂模式的时候不用加 new。
  • 加 new 的话是构造函数，工厂模式加 new 是寄生构造函数模式。
# 缺点
  • 无法识别为对象类型

# 构造函数模式

# 示例
function Cat (name, age) {
     this.name = name;
     this.age = age;
 }
 var cat1 = new Cat('小花','1');
 var cat2 = new Cat('小黑','2');

细节

  • 构造函数没有显示的创建对象。直接把方法和属性赋给了 this 对象。没有 return 语句

  • 构造函数就是函数，与普通函数的区别是通过 new 来调用

  • 构造函数名的首字母大写，与普通函数名作区别

优点

  • 构造函数的好处，是可以将它的实例化标识为特定的 Object 类型。

缺点

  • 每实例化一次构造函数，会把属性和方法再创建一遍。造成冗余。

# 原型模式

# 说明
  • 每个构造函数都有一个 prototype 属性，指向另一个对象。
  • 这个对象的所有属性和方法，都会被构造函数的实例继承。
  • 这意味着，我们可以把那些不变的属性和方法，直接定义在prototype对象上。
# 示例
function Cat (name,age) {
     this.name = name;
     this.age = age;
 }
 Cat.prototype.type = "猫";

var cat1 = new Cat('小花','1');
 var cat2 = new Cat('小黑','2');

for ( var key in cat1) {
     console.log(cat1[key])
 }
# 细节
  • 实例化的对象无法设置 prototype。
  • 实例化后的对象，无法用 console.log() 显示其 prototype 属性。
    需要通过 for ... in 才能显示对象中包含 prototype 的所有属性。
# 优点
  • prototype 的好处，是在实例化对象的时候，不会产生额外的内存。

# 原型模式的判断方法

isPrototypeOf()

  • 用于判断对象和实例之间的关系，是则 ture

  • Cat.prototype.isPrototypeOf(cat1)

hasOwnProperty()

  • 用于判断某变量名或者方法与实例对象的关系

  • cat1.hasOwnProperty("name")

in运算符

  • 用于判断某个实例是否含有某个属性

  • console.log( 'name' in cat1 )

  • 还可以用于遍历对象的所有属性

  • for ( var key in cat1 ) {}

# 动态原型模式

示例

function cat (name,age) {
     this.name = name;
     this.age = age;

if ( typeof this.sayName != "function" ) {
         cat.prototype.sayName = function () {
             console.log(this.name)
         }
     }
 }
 var cat1 = new cat('小花', 1);
 cat1.sayName()

细节

  • typeof 用于检查数据类型
  • prototype 模式可以写在构造函数里面
  • 动态原型，也可以写成动态this——this.sayName = functioin(){}
  • 在封装对象中写入判断，还可以在实例化之后用 instanceof 来检查它的类型。

优点

  • 在构造函数中加入判断，可以避免之后去写大堆的 if 检查每个属性和方法。

# 寄生构造函数模式

说明

  • 寄生构造函数模式，可为 Object、Array、Data 等原生构造函数增加所需的附加功能。
  • 如果想创建具有额外方法的数组，由于不能改变 Array 的原生构造函数，可以使用这种模式。

示例

function cat () {
    var cats = [];
    cats.push.apply(cats, arguments);
    cats.toPipetString = function () {
        return this.join("|");
    }
    return cats;
}

var cat1 = new cat('小红','小黑')

细节

  • 构造函数默认返回新对象实例。通过 return 可以重写函数返回值。
  • 在可以使用其他模式的情况下，不要使用这种模式。
  • 寄生构造函数模式返回的对象，与构造函数或者与构造函数的原型属性之间没有关系。
    构造函数返回的对象与在构造函数外部创建的对象没有什么不同。
    为此，不能依赖 instanceof 操作符来确定对象类型。

# 稳妥构造函数模式

说明

  • 在防止数据被其他应用程序（如Mashup程序）改动时使用
  • 有的安全环境会禁止使用 this 和 new
  • 稳妥构造函数遵循与寄生构造函数类似的模式。
    但有两点不同：一是新创建对象的实例方法不引用this；二是不使用new操作符调用构造函数。

示例

function cat (name) {
    var o = {};
    o.sayName = function () {
        console.log(name)
    }
    return o;
}
var cat1 = new cat('小花');
cat1.sayName()

细节

  • 没有 this、没有 new、要 return 对象
  • 属性加给构造函数里的对象
  • 使用稳妥构造函数模式创建的对象与构造函数之间没有关系，因此instanceof操作符对这种对象也没有意义

# 二. 继承

# 继承的原理

说明

  • JavaScript规定，任何函数都带有一个 .prototype (原型) 属性，指向原型对象。
    原型对象则带有一个 .constructor (构造器) 属性，这个 constructor 指向构造函数。
  • 凡是被 new 过的函数都叫构造函数。
  • 实例化后的函数，带有的是 .proto (原始) 属性。与 prototype 同样指向原型对象。

# 构造函数绑定

示例

     function A () { this.aa = 'aa' }

     function B () { 
         this.bb = 'bb'; 
         A.apply(this, arguments);
     }
     var c = new B();
     console.log(c.aa);
     console.log(c.bb);

细节

  • 通过 applay() 使 子类B 继承 父类A 的所有属性。

# 原型式继承

示例

function A () { this.aa = 'aa' }
function B () { this.bb = 'bb' }
B.prototype = new A ();
B.prototype.constructor = B;
var c = new B ();
console.log(c.aa)
console.log(c.bb)

# 封装原型式继承

示例

function f (child,parent) {
         let c = child,
             p = parent
         c.prototype = new p();
         c.prototype.constructor = c;
     }
     f(B,A);
     var c = new B;
     console.log(B.aa)
     console.log(B.bb)

# 拷贝继承

说明

  • 上面是采用 prototype 对象来实现继承，也可以用‘拷贝’的方式实现继承
  • 将父对象的所有属性和方法拷贝进子对象来实现继承。

示例

function Animal(){}
 Animal.prototype.species = "动物";
 function extend(Child, Parent) {
 var p = Parent.prototype;
     var c = Child.prototype;
     for (var i in p) {
     c[i] = p[i];
 }
     c.uber = p;
     }
 extend(Cat, Animal);
 var cat1 = new Cat("大毛","黄色");
 console.log(cat1.species); // 动物

# 三 . 非构造函数继承

# 说明
  • 非构造函数继承，既：不是构造函数，而是 Json格式。
  • 以下讲如何继承这种对象。

# object模式

示例

var Chiness = {
     nation: '中国'
 }
 function Object(o){
     function F () {}
     F.prototype = o;
     return new F();
 }
 var doctor = Object(Chiness);
 doctor.career = '医生';
# 细节
  • 创建空函数，把 prototype 指向被继承的父对象对象
  • 然后 new 给要继承的子对象。
  • 这样子对象就拥有了父对象的属性。

# 浅拷贝

# 说明
  • 除了使用 prototype(原型) 链以外，也可以把父对象的属性，通过 in运算符 拷贝给子对象。
# 示例
var Chiness = {
     nation: '中国'
 }
 function extendCopy (p){
     var c = {};
     for (var i in p) {
         c[i] = p[i]
     }
     return c;
 }
 var doctor = extendCopy(Chiness);
 doctor.career = '医生';
# 缺点
  • 如果父对象的属性等于数组或另一个对象，那么实际上，子对象获得的只是一个内存地址，而不是真正拷贝，因此存在父对象被篡改的可能。
  • 所以，extendCopy() 只是拷贝基本类型的数据，这种拷贝叫做 "浅拷贝"。这是早期 jQuery 实现继承的方式。

# 深拷贝

# 说明
  • 所谓 "深拷贝"，就是能够实现真正意义上的数组和对象的拷贝。只要递归调用 "浅拷贝" 就行了

示例

function deepCopy(p, c) { 
    var c = c || {}; 
    for (var i in p) { 
        if (typeof p[i] === 'object') { 
            c[i] = (p[i].constructor === Array) ? [] : {}; 
            deepCopy(p[i], c[i]); 
        } else { 
            c[i] = p[i]; 
        } 
    }
    return c;
}
